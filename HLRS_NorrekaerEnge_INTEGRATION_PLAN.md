# HLRS 与 Norrekaer Enge 数据集纳入框架详细规划

> 目标：
>
> 1. 将 `HLRS` 与 `Norrekaer Enge` 以**确定性逻辑**纳入现有 `STCWPF_final` 多数据集框架
> 2. 在保证数据接入准确的前提下，最大化体现：
>    - `warmup` 机制对新增风机的适应优势
>    - 基于真实空间坐标构建邻接矩阵的优势
> 3. 本文档只做规划与数据确定，不修改代码
>
> 约束：
>
> - 严格按照本文档最终定稿方案执行
> - 只采用已经实际读到、实际确认过的数据字段
> - 不补不存在的动态特征
> - 不对采样频率做模糊判断
> - 明确哪些风机可用于目标预测，哪些只能作为辅助/应剔除
> - 明确位置坐标是否可用于静态特征嵌入与邻接矩阵构建
> - 第一版不纳入任何“可选但不建议”的附加特征
> - 第一版不纳入 `elevation`，统一只使用可确认、可对应的平面坐标

# 1. 现有框架纳入新数据集时应关注的问题

参考 `Penmanshiel` / `Towie` 的接入经验，新增数据集时要明确以下内容：

1. **目标列是否存在且可直接对应 `Patv`**
2. **核心动态特征能否与现有框架语义对齐**
   - 优先：`Wspd / Wdir / Ndir / Pab1 / Pab2 / Pab3 / Patv`
   - 其次：温度、密度、场级背景气象等
3. **风机坐标是否准确、是否能和风机编号一一对应**
4. **采样频率是否固定；若不完全固定，主频是多少，是否可重采样统一**
5. **数据时间跨度是否足够长，能支持 pretrain / val / streaming test**
6. **风机数量是否足够，并且是否存在“有特征但无功率”的风机**
7. **如何设计初始风机与后续分批加入风机，才能体现 warmup 与邻接矩阵的优势**
8. **是否存在明显不合理值，需要在评估时 mask 掉，防止指标失真**

# 3. 是否纳入 `elevation` 的客观结论

## 3.1 结论

**第一版不建议把 `elevation` 纳入统一框架。**

## 3.2 原因

### 原因 1：跨数据集并不一致

当前能够明确确认逐机高程信息的，实际上只有 `sdwpf`。对于：

- `HLRS`
- `Norrekaer Enge`
- 以及此前部分其他数据集

都没有同等质量、逐机一一对应、可直接读取的高程列。

如果为了统一维度而：

- 对部分数据集填 0
- 或填一个场级常数

这在工程上虽然能做，但在实验叙事上并不理想，因为这会造成：

- 有的数据集该维度是真实几何信息
- 有的数据集该维度只是占位常数

这会让同一个静态通道的语义不一致。

### 原因 2：常数高程几乎不提供有效区分信息

若某个数据集所有风机 `elevation` 都相同，那么这一列在风机间没有区分度：

- 对邻接矩阵构建基本没有帮助
- 对静态嵌入区分风机几乎没有帮助
- 对模型更像是一个常数偏置项

因此它带来的收益通常极小。

### 原因 3：还可能引入轻微噪声或语义污染

如果把：

- 真实高程
- 常数 0
- 常数 -3

混在同一个统一静态特征维度里，模型会接收到一个“表面统一、实际含义不统一”的输入通道。

这类做法通常不会带来很大收益，反而可能：

- 稀释静态嵌入的有效信息密度
- 增加不同数据集之间的分布差异
- 让实验解释变差

## 3.3 客观建议

从**严谨性、跨数据集一致性、实验可解释性**来看：

- **第一版统一去掉 `elevation` 更合理**
- 静态特征只保留真实且可靠的 `x/y`

## 3.4 对实验效果的实际影响判断

客观地说：

- 去掉 `elevation`，大概率**不会明显伤害实验效果**
- 但能明显提升方案的严谨性与可解释性
- 在目前这几个数据集里，真正关键的静态信息主要还是**平面空间位置关系**，也就是 `x/y`

因此第一版推荐：

- **邻接矩阵：只基于 `x/y` 构建**
- **静态嵌入：只基于 `x/y` 构建**

## 5.6 明显不合理值与评估 mask 策略

HLRS 不采用 `sdwpf` 那套严格论文规则，但需要有自己的轻量异常值逻辑，避免评估指标被明显脏值拖垮。

### 评估时需要 mask 的情况

1. `Patv` 缺失
2. `Wspd` 缺失
3. `Patv < 0`
4. `Wspd < 0`
5. `Patv` 明显超出机组额定功率合理范围
6. 必要背景列缺失：
   - `Wdir`
   - `Wspd_w`
   - `T2m`

### 处理原则

- **训练前不做过度激进过滤**
- **评估时依赖 mask 机制排除明显不合理样本**
- 与你当前代码中的 mask 指标逻辑对接即可，不需要再单独发明一套复杂规则

这样做的目的不是模仿 `sdwpf`，而是：

- 保住有效样本量
- 防止少量脏值把指标拉坏
- 保持新增数据集处理方式的可解释性

## 5.7 推荐初始风机与扩容方案

为了体现 warmup 与邻接矩阵优势，HLRS 不建议一开始就把 200 台全部放入 pretrain。

### 原则

1. 初始风机应形成一个**空间上相对连续的核心区域**
2. 后续加入应按**空间外扩**方式进行，而不是随机加入
3. 这样更能体现：
   - 新增风机进入后，warmup 如何快速利用已有结构知识
   - 邻接矩阵如何帮助从近邻传播信息

### 推荐方案

- 初始风机数：`80`
- 扩容次数：`4`
- 每次新增：`30`

即：
- Stage 0：80 台
- Stage 1：110 台
- Stage 2：140 台
- Stage 3：170 台
- Stage 4：200 台

### 风机加入策略

建议根据 `farm.yaml` 坐标做以下顺序：

#### 方案：按空间中心向外扩张
1. 先计算全场几何中心
2. 按风机到中心距离排序
3. 距离最近的 80 台作为初始风机
4. 后续按距离由近到远分批加入

### 为什么这样设计最好

- 初始图结构更紧凑，预训练更稳定
- 新增风机会处在旧风机外圈，warmup 更有意义
- 更能体现基于坐标构图的邻接矩阵优势

## 6.5 缺测值与明显不合理值处理策略

这里按照你当前代码已有的 mask 指标逻辑来设计，不额外发明复杂规则。

### 6.5.1 缺测值处理

1. 先建立完整 10 分钟时间索引
2. 所有风机与桅杆信号重建到该时间轴
3. 对缺测值保留 `NaN`
4. 评估时通过现有 mask 逻辑排除无效位置

### 结论

- **保留 `NaN` 是合理的**
- 因为你现有评估链路本来就有 mask 机制
- 这样比强行插值所有变量更稳妥，也更符合该数据集实际情况

---

### 6.5.2 明显不合理值处理

Norre 不采用 `sdwpf` 那种严格论文清洗规则，但需要加入自己的轻量异常值判断，尤其在评估时 mask 掉，防止指标崩溃。

### 评估时需要 mask 的情况

1. `Patv` 缺失
2. `Wspd` 缺失
3. `Ndir` 缺失
4. `Wdir` 缺失
5. `Patv < 0`
6. `Wspd < 0`
7. `Patv` 明显大于机组额定功率合理上界
8. `Ndir` / `Wdir` 出现明显异常角度值

### 处理原则

- 训练阶段不过度删除样本
- 评估阶段通过 mask 排除明显不合理值
- 与现有指标计算接口直接衔接

### 为什么这样做合适

- 该数据集没有像 `sdwpf` 那样现成且权威的严格论文规则
- 过于激进的过滤可能损失大量样本
- 完全不处理又会让少量脏值破坏指标稳定性

因此最合适的是：

- **轻量异常值规则 + 评估 mask**

---

## 6.6 时间跨度

- `1991-12-22 ~ 1993-06-27`
- 约 **1.5 年**

### 结论

时间跨度完全足够支持：
- pretrain
- validation
- streaming test

## 6.7 推荐初始风机与扩容方案

为了更好体现 warmup 机制和邻接矩阵的优越性，Norre 非常适合做“行列式阵列逐步扩容”。

### 原则

1. 初始风机应形成一块连续子阵列
2. 后续按空间外圈逐步加入
3. 这样新增风机与旧风机存在明确邻接关系，最能体现 warmup 的价值

### 推荐方案

针对 40 台可预测风机：

- 初始风机数：`16`
- 扩容次数：`3`
- 每次新增：`8`

即：
- Stage 0：16 台
- Stage 1：24 台
- Stage 2：32 台
- Stage 3：40 台

### 风机加入策略

建议按照坐标整理后，按“阵列核心 -> 中圈 -> 外圈”加入。

如果坐标整理后保留 `a2~f7` 这样的网格命名，可优先：

#### 初始风机
选择几何中心附近的一块连续子阵列，例如中部 4x4

#### 第一次扩容
加入与核心区相邻的一圈风机

#### 第二次扩容
再加入更外围一圈

#### 第三次扩容
加入最边缘风机

### 为什么这种设计好

- 新增风机总能在图上找到相邻已知风机
- warmup 有真实空间先验可利用
- 更容易体现邻接矩阵相比纯全连接/无图结构的优势

- 可以纳入
- 缺失功率列的 `a1`、`f6` 直接排除
- 第一版只用 `Wspd/Wdir/Ndir/Patv`
- 使用完整时间范围 `1991-12-22 ~ 1993-06-27`
- 对缺测值保留 `NaN`，评估时结合现有 mask 逻辑处理
- 再叠加轻量明显异常值 mask，防止指标崩溃
- 静态特征只用 `x/y`

这套方案的重点不是“把所有能读到的列都塞进来”，而是：

- **用最确定、最可解释的特征把数据集准确纳入**
- **把实验重点集中在 warmup 机制与空间邻接矩阵优势上**
- **避免无必要的附加特征和伪静态信息稀释实验结论**
